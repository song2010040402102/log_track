条件表达式

日志数据在查询过程中由于没有算数及赋值运算的必要性，所以只需要支持关系和逻辑运算即可。由于逻辑非和括号都可以通过运算符转化或分解来完成，所以运算符仅需要支持6种关系运算和与或逻辑运算，并保证优先级：关系>与>或即可。为了简化条件表达式，仅支持整数、小数、字符串三种数据类型，并添加数据集的支持，其中字符串不需要加引号，但区分大小写，只有表头字段不区分大小写，数据集无非就两种情况，一个是离散型，一个是区间型，对于区间型只支持数值。以下则为条件表达式的语法及demo的简要介绍

  • 关系: >= > <= < != =
  • 逻辑: && ||
  • 数据集: 离散型和区间型
  • 离散型: 单值和多值，多值之间用逗号隔开
  • 区间型: [] [) (] ()，同样用逗号隔开
  • 数据类型: 整数 小数 字符串，其中字符串不支持大小比较
  • 健名称不区分大小写，支持空格和TAB
  • 例1：uid=6419357,6420066 && platform=wechat || regTime>1567560586 && offlineTime=[100,1000]
  • 例2：uid!=6419357,6420066 && platform!=wechat || regTime<=1567560586 && offlineTime!=(100,1000]
